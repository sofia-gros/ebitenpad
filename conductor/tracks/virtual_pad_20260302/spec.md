# 仕様書: バーチャルスティックとボタンの実装

## 概要
このトラックでは、タッチスクリーンやマウスで操作可能なバーチャル UI（スティック、ボタン）を実装します。これにより、モバイルデバイスや Web 環境において直感的な操作が可能になり、既存の `Action` システムと統合されます。

## ユーザーストーリー
- 開発者として、スマホ画面上に「ジャンプ」ボタンや「移動」スティックを配置したい。
- 開発者として、バーチャル UI の入力を既存の論理アクションにバインドしたい。
- ユーザーとして、マルチタッチでスティックとボタンを同時に操作したい。

## 要件

### 1. バーチャルボタン (`virtual/button.go`)
- `Button` 構造体: 位置、半径、現在の押下状態、タッチ ID ロック。
- `SetPosition(x, y float64)`, `SetRadius(r float64)`.
- 描画機能: 円形または矩形の簡易的なデバッグ描画。

### 2. バーチャルスティック (`virtual/stick.go`)
- `Stick` 構造体: 中心位置、半径、現在の入力ベクトル (x, y)、タッチ ID ロック。
- `SetPosition(x, y float64)`, `SetRadius(r float64)`.
- 入力の正規化: 半径内での位置を -1.0 から 1.0 に変換。

### 3. バーチャル UI マネージャー (`virtual/virtual.go`)
- `VirtualPad` 構造体: ボタンとスティックのリストを管理。
- `Update()`: Ebitengine の `AppendTouchIDs` を使用して各コンポーネントの状態を更新。

### 4. 入力統合 (`input/input.go`)
- `Input.BindButton(action Action, button *virtual.Button)`
- `Input.BindStick(action Action, stick *virtual.Stick)`
- `Input.Update()` 内でバーチャル UI の入力をポーリングし、アクション状態を更新。

## 受入条件
- [ ] バーチャルボタンをタッチ/クリックしてアクションが発火すること。
- [ ] バーチャルスティックで 360 度方向のベクトル入力が得られること。
- [ ] マルチタッチで複数のバーチャルコンポーネントが干渉なく動作すること。
- [ ] WASM 例にバーチャルボタンが表示され、操作できること。
